# 万能导入 V2 — 格式知识库

本文档描述常见物流单据的格式特征，供 AI 生成解析规则时参考。

---

## 1. 配送发货单（Excel 标准格式）

### 文件特征
- 文件名通常包含"配送发货单"
- 42 列，表头在第 4 行（0-based 索引 3）
- 单据编号格式：`PS` 开头 + 日期 + 序号

### 布局结构
```
Row 0: 标题（黎明屯铁锅炖配送中心-配送发货单PS...）
Row 1: 机构信息（收货机构、供货机构、送货机构、订货机构）
Row 2: 状态信息（发货操作时间、复审时间、单据状态、分拣状态）
Row 3: 表头（序号、物品分类、物品编码、物品名称...42列）
Row 4+: 数据行
       ...
Row N-3: 合计行
Row N-2: 单据号信息（单据号、上游单据、创建日期）
Row N-1: 收货人信息（收货人、收货电话、收货地址）
Row N:   备注信息
```

### 典型表头列名
| 列名 | 说明 |
|------|------|
| 序号 | 行号 |
| 物品分类 | 食材、工服等分类 |
| 物品编码 | SKU 编码 |
| 物品名称 | 商品名称 |
| 规格型号 | 规格描述 |
| 存储方式 | 常温、冷藏、冷冻 |
| 分拣状态 | 无需分拣、待分拣 |
| 订货单位 | 箱、件、袋 |
| 订货数量 | 订货数量 |
| 发货数量 | 发货数量 |
| 发货单价/金额 | 价格信息 |
| 发货仓库 | 仓库名称 |
| 基准单位 | 最小单位 |
| 备注 | 备注 |

### 收货人信息提取
收货人信息在文件尾部，需要用 `tailExtract` 提取：
- `收货人：` → receiverName
- `收货电话：` → receiverPhone
- `收货地址：` → receiverAddress

### 推荐操作顺序
```
skipRows(4) → filterEmptyRows → skipTotalRows → aggregateBy(序号, first, [物品分类, 物品编码, 物品名称, 规格])
```

## 2. 多门店分Sheet出库单

### 文件特征
- 每个 Sheet 对应一个门店
- Sheet 名 = 门店名
- 8 列标准格式

### 布局结构
```
Row 0: 标题（尹三顺自助烤肉（银泰店）出库单）
Row 1: 基本信息（出库日期、仓库、配送方式、打印时间）
Row 2: 空行
Row 3: 空行
Row 4: 表头（序号、物品编码、物品名称、规格型号、单位、出库数量、仓库、备注）
Row 5+: 数据行
Row N-4: 合计行
Row N-3: 收货门店信息
Row N-2: 联系电话/收货地址
Row N-1: 空行
Row N:   制单信息（制单人、审核人、签字）
```

### 底部信息（每个Sheet独立）
```
收货门店：银泰店
联系人：王店长
联系电话：13900001111
收货地址：汉口解放大道688号银泰百货B1层
```

### 推荐操作
```json
{
  "type": "multiSheet",
  "perSheetTailExtract": [
    { "marker": "收货门店：", "targetField": "storeName" },
    { "marker": "联系人：", "targetField": "receiverName" },
    { "marker": "联系电话：", "targetField": "receiverPhone" },
    { "marker": "收货地址：", "targetField": "receiverAddress" }
  ]
}
```

## 3. 库存查询单

### 文件特征
- 文件名包含"查询结果"或"库存"
- 列中包含门店分货列（门店名为列维度）
- 可能含公式列

### 布局结构
```
Row 0: 表头（仓库名称、货主名称、SKU名称、SKU条码、银泰、金桥、金银潭...）
Row 1+: 数据行
```

### 典型表头
| 列名 | 说明 |
|------|------|
| 仓库名称 | 仓库名 |
| 货主名称 | 货主 |
| SKU名称 | 商品名称 |
| SKU条码 | 条码 |
| 外部商品编码 | 外部编码 |
| 库存状态 | 正常、冻结 |
| 在库数量的总和 | 总库存 |
| 可用数量的总和 | 可用库存 |
| 银泰/金桥/金银潭 | 各门店分货数量 |

## 5. 门店调拨单（卡片式）

### 文件特征
- 含"调拨记录 #N"标识
- 每个调拨记录为一个卡片

### 布局结构
```
Row 0: 标题（武汉配送中心 · 门店调拨单）
Row 1: 基本信息（调拨单号、调出仓库、调拨日期）
Row 2: 空行
Row 3: 空行
Row 4: ▶ 调拨记录 #1
Row 5: 调入门店 | xxx | 收货人 | xxx | 电话 | xxx
Row 6: 收货地址 | xxx
Row 7: 物品编码 | 物品名称 | 规格 | 数量
Row 8+: 物品数据行
Row N: 空行
Row N+1: ▶ 调拨记录 #2
...
```

### 推荐操作
```json
{
  "type": "cardBoundary",
  "startMarker": "▶ 调拨记录 #(\\d+)",
  "metaRowPatterns": [
    { "pattern": "调入门店.*?\"([^\"]+)\"", "targetField": "storeName" },
    { "pattern": "收货人.*?\"([^\"]+)\"", "targetField": "receiverName" },
    { "pattern": "电话.*?\"([^\"]+)\"", "targetField": "receiverPhone" },
    { "pattern": "收货地址.*?\"([^\"]+)\"", "targetField": "receiverAddress" }
  ],
  "cardHeaderMarker": "物品编码"
}
```

## 6. 配送发货单（PDF）

### 文件特征
- PDF 格式，2 页
- 含分页信息"第N页 / 共M页"
- 基本信息在开头，物品在中后部

### 布局结构
```
Page 1:
- 标题（店名-配送单）
- 基本信息区
  单据编号：PS2604210007
  单据状态：已发货
  收货机构：xxx
  订货机构：xxx
  供货机构：xxx
  ...
- 物品表头（物品类别、物品编码、物品名称、规格型号、订货单位、发货数量）
- 数据行（分页）
- Page N / 共 M 页

Page 2 (末页):
- 续数据行
- 合计行
- 制单信息（制单日期、创建人、发货人）
- 收货人信息（收货人、收货电话、收货地址）
```

### 收货人信息（末页）
```
制单日期：2026/04/21 14:10:19
创建人：陈燕
发货人：张慧
收货人：荣丽
收货电话：13130093946
收货地址：辽宁省鞍山市铁东区...
```

### 推荐操作
```json
{
  "type": "headerExtract",
  "rules": [
    { "marker": "单据编号：", "targetField": "externalCode" },
    { "marker": "收货机构：", "targetField": "storeName" },
    { "marker": "收货人：", "targetField": "receiverName" },
    { "marker": "收货电话：", "targetField": "receiverPhone" },
    { "marker": "收货地址：", "targetField": "receiverAddress" }
  ]
}
```

## 附录：标准字段说明

### A 组字段（收货门店）
至少填写 A 组或 B 组之一
- `storeName`: 收货门店/收货机构/调入门店

### B 组字段（收件人）
- `receiverName`: 收件人姓名
- `receiverPhone`: 收件人电话
- `receiverAddress`: 收件人地址

### SKU 必填字段
- `skuCode`: SKU 物品编码（必填）
- `skuName`: SKU 物品名称（必填）
- `skuQuantity`: SKU 发货数量（必填，正数）
- `skuSpec`: SKU 规格型号（选填）
- `remark`: 备注（选填）

### 其他字段
- `externalCode`: 外部编码/配送单号/单据号（选填）
